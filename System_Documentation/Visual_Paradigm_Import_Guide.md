# Visual Paradigm Import Guide
## TechnoPath: FDD & UCD Diagrams

---

## Overview

Since `.vpp` files are proprietary binary files generated by Visual Paradigm software, this guide provides alternative methods to import the Functional Decomposition Diagram (FDD) and Use Case Diagram (UCD) into Visual Paradigm.

---

## Method 1: PlantUML Import (Recommended)

Visual Paradigm supports importing PlantUML files directly.

### Steps for FDD:
1. Open Visual Paradigm
2. Go to **Tools** → **PlantUML** → **Import PlantUML...
3. Select file: `FDD_TechnoPath_Functional_Decomposition.puml`
4. Choose diagram type: **Component Diagram** or **Package Diagram**
5. Click **Import**

### Steps for UCD:
1. Open Visual Paradigm
2. Go to **Tools** → **PlantUML** → **Import PlantUML...
3. Select file: `UCD_TechnoPath_Use_Case_Diagram.puml`
4. Choose diagram type: **Use Case Diagram**
5. Click **Import**

## Method 2: Manual Creation Guide

### Creating FDD in Visual Paradigm:

1. **Create New Project**
   - File → New Project → Name: "TechnoPath_FDD"

2. **Create Package Diagram** (for functional decomposition)
   - Diagram → New → Package Diagram

3. **Create Hierarchy**:

   │   ├── Sub-package: "Notifications System"
   │   ├── Sub-package: "Information Center"
   │   └── Sub-package: "User Engagement"

   ├── Package: "Admin Panel Module" (Color: Light Green)
   │   ├── Sub-package: "Admin Dashboard"
   │   ├── Sub-package: "Map Management"
   │   │   ├── "Facilities Management"
   │   │   ├── "Rooms Management"
   │   │   ├── "Navigation Graph"

   │   │   └── "QR Codes"

### Creating UCD in Visual Paradigm:

1. **Create New Project**
   - File → New Project → Name: "TechnoPath_UCD"

2. **Create Use Case Diagram**
   - Diagram → New → Use Case Diagram

3. **Add Actors** (with stereotypes):
   ```
   Actor: "Super Admin (Safety & Security)" → Stereotype: <<SuperAdmin>>
   Actor: "Dean (College Dean)" → Stereotype: <<Dean>>
   Actor: "Program Head (College Program)" → Stereotype: <<ProgramHead>>
   Actor: "Basic Ed Head (Basic Education)" → Stereotype: <<BasicEdHead>>
   Actor: "Guest User (Students, Visitors, Faculty)" → Stereotype: <<Guest>>
   ```

4. **Add Use Cases by Package**:

   **Map Management Package**:
   - UC: "Manage Facilities" → Actor: Super Admin
   - UC: "Manage All Rooms" → Actor: Super Admin
   - UC: "Manage Own Dept Rooms" → Actors: Dean, Program Head, Basic Ed Head
   - UC: "Manage Navigation Graph" → Actor: Super Admin
   - UC: "Manage FAQ/Chatbot" → Actor: Super Admin
   - UC: "Generate QR Codes" → Actor: Super Admin

   **Communications Package**:

   - UC: "Post Dept Announcement" → Actor: Dean
   - UC: "Approve Announcements" → Actor: Super Admin
   - UC: "Send Notifications" → Actor: Super Admin

   **Administration Package**:
   - UC: "Manage Admin Accounts" → Actor: Super Admin
   - UC: "View All Feedback" → Actor: Super Admin
   - UC: "View Dept Feedback" → Actor: Dean
   - UC: "View Audit Log" → Actor: Super Admin

   **Mobile User Features Package**:
   - UC: "View Campus Map" → Actor: Guest
   - UC: "Search Locations" → Actor: Guest
   - UC: "Navigate Campus" → Actor: Guest
   - UC: "Use Chatbot" → Actor: Guest
   - UC: "View Notifications" → Actor: Guest

5. **Add Relationships**:

## Method 3: XMI Export/Import

Visual Paradigm supports XMI format for UML model interchange.

### To import XMI:
1. File → Import → XMI...
2. Select XMI file
3. Choose import options
4. Click Import

## Method 5: API/Script Import

For advanced users, Visual Paradigm provides OpenAPI and scripting support:

### Groovy Script Example:
```groovy
// Create actor
def actor = modelManager.createActor("Super Admin")
actor.addStereotype("SuperAdmin")

// Create use case
def useCase = modelManager.createUseCase("Manage Facilities")
useCase.addActor(actor)

// Create relationship
def relationship = modelManager.createRelationship(useCase, actor, "Association")
```

## Visual Paradigm Recommended Settings

### For FDD (Package Diagram):
- **Diagram Type**: Package Diagram
- **Show Package Contents**: Yes
- **Color Scheme**: 
  - Mobile Module: #E3F2FD (Light Blue)
  - Admin Module: #E8F5E9 (Light Green)
  - Core Module: #FCE4EC (Light Pink)
- **Layout**: Hierarchical
- **Orientation**: Top-to-Bottom

### For UCD (Use Case Diagram):
- **Diagram Type**: Use Case Diagram
- **Actor Style**: Stick Figure or Icon
- **Show Multiplicity**: Yes
- **Color Scheme**:
  - Super Admin: #FFCDD2 (Red-tinted)
  - Dean: #C8E6C9 (Green-tinted)
  - Program Head: #BBDEFB (Blue-tinted)
  - Basic Ed Head: #FFECB3 (Yellow-tinted)
  - Guest: #F5F5F5 (Gray)
- **Layout**: Auto-route
- **Stereotype Display**: Icon + Label

## Troubleshooting

### PlantUML Import Issues:
- **Error**: "Invalid PlantUML syntax"
  - **Fix**: Ensure PlantUML plugin is installed (Help → Install New Software)
  
- **Error**: "Diagram type not supported"
  - **Fix**: Convert to supported diagram type after import

### Performance Tips:
- For large diagrams, disable auto-layout before importing
- Enable "Simplify Complex Diagrams" option
- Use layers to organize complex use case diagrams

1. **Review and Adjust Layout**

   - Tools → Doc → Composer
