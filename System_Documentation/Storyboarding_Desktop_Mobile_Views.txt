4. SCREEN FLOW DESCRIPTIONS
================================================================================

4.1 GUEST USER FLOW (Mobile)
----------------------------

Entry Point: Splash Screen
The user first sees the Splash Screen with SEAIT logo, app name, and welcome message.
From here, two paths exist:

Path A: Tutorial Flow
1. User taps "Get Started"
2. App shows onboarding tutorial screens (if implemented)
3. Tutorial explains features: Map, Navigation, Chatbot
4. User completes tutorial and lands on Home

Path B: Skip Tutorial
1. User taps "Skip Tutorial"
2. App directly navigates to Home screen

Main Navigation Hub: Home Screen
The Home screen serves as the central hub with multiple navigation options:

Option 1: Menu Exploration
- User taps Menu button
- Slide-up menu appears
- User selects: Building Info, Rooms Info, Instructor Info, Employees, or Laboratory Info
- System displays selected information view

Option 2: Search
- User taps search bar
- Search interface expands
- User types query
- Search results displayed
- User selects result
- Navigate to location details or Navigation view

Option 3: Map Interaction
- User interacts with map markers
- Marker popup appears
- Popup shows location name and quick actions
- User can navigate from popup

Option 4: Direct Navigation
- User taps Navigate button in bottom bar
- App opens Navigation view
- User enters start and end points
- App calculates and displays route

Secondary Features Access:
From Home, user can access:
- Chatbot: For AI-assisted queries
- Notifications: For campus announcements
- Settings: For app preferences
- Rate & Feedback: For user input

Navigation Result:
When route is found:
- Map shows highlighted path
- Directions panel slides up
- Turn-by-turn instructions displayed
- User can follow directions to destination


4.2 ADMIN USER FLOW (Desktop)
-----------------------------

Entry Point: Admin Login
- Admin accesses /admin-login URL
- Login form displayed with username/password fields
- Authentication via JWT tokens
- On success, redirected to Admin Dashboard

Main Hub: Admin Dashboard
The Dashboard provides overview and navigation to all admin functions:

Map Management Branch:
1. Facilities Management
   - View all buildings in table
   - Add new facility
   - Edit existing facility details
   - Soft delete facility

2. Rooms Management
   - View rooms by facility
   - Add rooms to buildings
   - Edit room details (floor, type, capacity)
   - Remove rooms

3. Navigation Graph
   - View navigation nodes (waypoints, entrances, rooms)
   - Add/edit navigation nodes with coordinates
   - Manage edges (connections between nodes)
   - Set distances and bidirectional flags

4. FAQ Management
   - View existing FAQ entries
   - Add new Q&A pairs
   - Organize by category
   - Set keywords for search

5. QR Code Generation
   - Generate QR codes for buildings
   - Generate QR codes for rooms
   - Download QR code images
   - Print batch QR codes

Communications Branch:
1. Announcements
   - Create new announcement
   - Set scope (campus-wide, department, etc.)
   - Submit for approval (if not Super Admin)
   - View announcement history

2. Pending Approvals
   - View announcements awaiting approval
   - Review content and scope
   - Add reviewer notes
   - Approve and publish, or reject
   - Rejected items returned to sender

3. Send Notifications
   - Compose notification message
   - Select target audience
   - Set priority level
   - Send immediately or schedule

Administration Branch:
1. Admin Accounts
   - View all admin users
   - Create new admin accounts
   - Assign roles (Super Admin, Dean, Program Head, Basic Ed Head)
   - Set department associations
   - Deactivate accounts

2. Feedback Review
   - View user feedback submissions
   - See ratings and comments
   - Filter by facility or room
   - Flag inappropriate content

3. Audit Log
   - View system audit trail
   - Filter by admin user
   - Filter by action type
   - Filter by date range
   - Export audit data

System Branch:
1. View Site
   - Opens main application in new tab
   - Allows admin to see user view

Workflow: Announcement Approval
1. Dean/Program Head creates announcement
2. System flags for approval (if required)
3. Super Admin sees pending item
4. Super Admin reviews content
5. Decision:
   - Approve: Published immediately, notification sent
   - Reject: Returned to creator with notes
6. Audit log records action

Workflow: Facility Management
1. Super Admin adds new building
2. Enters: Code, Name, Department, Floors, Coordinates
3. System creates facility record
4. Admin adds rooms to facility
5. Admin creates navigation nodes for building
6. Admin connects nodes with edges
7. Facility appears on public map
